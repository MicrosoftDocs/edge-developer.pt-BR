# <a name="javascript-representation-of-windows-runtime-types"></a>Representação em JavaScript dos tipos de Windows Runtime  

[!INCLUDE [deprecation-note](../includes/legacy-edge-note.md)]  

A tabela a seguir descreve a maneira como o JavaScript representa os tipos do Windows Runtime.  

> [!IMPORTANT]
> Os recursos do Windows Runtime não estão disponíveis para aplicativos executados no Internet Explorer.  

## <a name="windows-runtime-types-in-javascript"></a>Tipos do Windows Runtime em JavaScript  

| Tipo do Windows Runtime | Representação javascript | Descrição |  
|:--- |:--- |:--- |  
| `UInt8` | `Number` | Um Windows Runtime `Uint8` é um inteiro de 8 bits não assinado, representado como um número JavaScript.  Um valor JavaScript é convertido primeiro em um número JavaScript e, em seguida, `ToUint32` o processo é seguido.  Se o valor do número JavaScript estiver fora do intervalo de um Uint8, o resultado terá o modulo 2^8 aplicado.  |  
| `Int32` | `Number` | Um Windows Runtime é um inteiro assinado de `Int32` 32 bits, representado como um número JavaScript.  Um valor JavaScript é convertido primeiro em um número JavaScript e, em seguida, `ToInt32` o processo é seguido.  Se o valor do número JavaScript estiver fora do intervalo de um , o resultado terá `Int32` o modulo 2^16 aplicado.  |  
| `Int64` | `Number` | Um Windows Runtime é um inteiro assinado de 64 bits, representado como um número padrão se estiver dentro do `Int64` intervalo \[-2^53, 2^53\].  Se ele estiver fora desse intervalo, ele será representado como um valor de número que tem um armazenamento de backing personalizado que mantém os 64 bits completos de dados inteiros.  Todas as operações matemáticas nesses valores de número int64 personalizados causam a conversão do valor em um número padrão no intervalo \[-2^53, 2^53\].  Se o valor estiver fora desse intervalo, será gerado um erro de tipo.  As exceções a esse processo de conversão são as operações , , e, que comparam os argumentos com base nos 64 bits completos quando passados dois valores representados `==` `===` de `SameValue` `RelationalComparison` 64 bits.  Se um dos argumentos for um número JavaScript padrão, essas operações também converterão o argumento em um número JavaScript antes da comparação.  Um valor JavaScript é atribuído diretamente se for um valor Int64 em si; caso contrário, o resultado da aplicação `ToInteger` da conversão no valor é passado para o Windows Runtime.  Se o tipo de coerção falhar, uma exceção será retornada.  |  
| `Uint64` | `Number` | Um Windows Runtime é um inteiro de 64 bits não assinado, representado como um número padrão se ele estiver dentro do intervalo `Uint64` \[0, 2^53\].  Se ele estiver fora desse intervalo, ele será representado como um número que tem um armazenamento de backing personalizado que mantém os 64 bits completos de dados inteiros não assinados.  Todas as operações matemáticas nesses valores Uint64 personalizados causam a conversão do valor em um número padrão no intervalo \[0, 2^53\].  Se o valor estiver fora desse intervalo, será gerado um erro de tipo.  As exceções a esse processo de conversão são as operações , , e, que comparam os argumentos com base nos 64 bits completos quando passaram dois valores de `==` `===` `SameValue` `RelationalComparison` 64 bits.  Se um dos argumentos for um número JavaScript padrão, essas operações também converterão o argumento em um número JavaScript antes da comparação.  Um valor JavaScript será atribuído diretamente se for um valor Uint64 em si; caso contrário, o resultado da aplicação da conversão no valor, seguido de tomar `ToInteger` o modulo 2^52, será passado para o Windows Runtime.  Se a conversão de tipo falhar, uma exceção será retornada.  |  
| `Single` | `Number` | Um Windows Runtime é um número de ponto flutuante de 32 bits, representado como um número JavaScript escolhendo o valor de precisão dupla mais próximo ao valor de `Single` precisão única.  Um valor JavaScript é convertido em um número JavaScript e, em seguida, o intervalo verificado para garantir que o valor possa ser representado em um número de ponto flutuante IEEE 754 de precisão única.  Se a verificação de conversão ou intervalo falhar, um erro de empacotamento será retornado.  |  
| `Double` | `Number` | Um Windows Runtime é um número de ponto flutuante de `Double` 64 bits.  `ToNumber` é usado para converter um Windows Runtime `Double` em um número JavaScript.  Se a conversão falhar, um erro de empacotamento será retornado.  |  
| `Boolean` | `Boolean` | Um Tempo de Execução do Windows é um valor de 8 bits onde zero é e qualquer valor diferente de zero é , representado `Boolean` `false` como um `true` JavaScript `Boolean` .  Um valor JavaScript é convertido pela primeira vez em um JavaScript `Boolean` por `ToBoolean` .  Isso significa que uma função do Windows Runtime declarada como `void func(BOOL);` poderia ser escrita em JavaScript como `obj.func("test");` .  A função Tempo de Execução do Windows é chamada com o `BOOL` parâmetro passado como `TRUE` .  Se a conversão falhar, um erro de empacotamento será retornado.  |  
| `Char16` | `String` | Um Windows Runtime é um caractere Unicode de 16 bits, representado como uma cadeia de `Char16` caracteres JavaScript que contém um único caractere.  Um valor JavaScript é convertido em uma cadeia de caracteres JavaScript e, em seguida, verificado para garantir que a cadeia de caracteres seja `ToString` apenas um único caractere longo.  O caractere único é passado como o `Char16` valor.  Se a verificação de conversão ou comprimento falhar, um erro de empacotamento será retornado.  |  
| `String` | `String` | Um Windows Runtime `String` é uma sequência imutável de `Char16` objetos.  As cadeias de caracteres são representadas como objetos `String` JavaScript.  As cadeias de caracteres do Windows Runtime não têm valores diferentes e a cadeia de `null` caracteres vazia \(""\).  `null` e valores de cadeia de caracteres vazios são convertidos para a cadeia de caracteres vazia JavaScript.  **OBSERVAÇÃO**: quando JavaScript é passado para um parâmetro do Windows Runtime que espera uma cadeia de caracteres, ele produz o valor de cadeia de caracteres "null", não ou a cadeia de `null` caracteres `null` vazia \(""\).  As cadeias de caracteres passadas para o Windows Runtime sempre devem ser instauidas para a cadeia de caracteres vazia.  |  
| `Enumeration` | `Object` | Um Windows Runtime é um inteiro de 32 bits \(assinado ou não assinado\) que tem um conjunto `Enumeration` associado de constantes nomeadas.  Em JavaScript, uma enumeração é representada como um objeto que contém um campo somente leitura para cada valor nomeado.  Os valores de enumeração são convertidos em números JavaScript conforme definido anteriormente `Int32` para `UInt32` e .  Quando um valor JavaScript é convertido em uma enumeração do Windows Runtime, ele é convertido, truncado e marcado como descrito anteriormente.  No entanto, o valor resultante não é verificado em relação aos valores nomeados definidos especificados na enumeração.  |  
| `Structure` | `Object` | Um Windows Runtime `Structure` é uma coleção heterogênea de campos de dados nomeados.  Em JavaScript, uma estrutura é representada como um objeto JavaScript que contém uma propriedade nomeada para cada campo na estrutura.  Se a conversão de qualquer valor de estrutura falhar, um erro de empacotamento será retornado.  Campos no objeto JavaScript que não têm equivalentes na estrutura do Windows Runtime são ignorados.  **OBSERVAÇÃO**: uma estrutura do Windows Runtime não pode ser instaurou em JavaScript com a `new` palavra-chave.  |  
| `Array` | `Array` | Um Windows Runtime `Array` é convertido em uma matriz JavaScript.  No entanto, as matrizes do Windows Runtime não são resizáveis, portanto, algumas operações de matriz JavaScript não são possíveis.  A propriedade interna de uma matriz convertida não está definida como "Matriz" porque isso não é permitido pela especificação `[[Class]]` ECMAScript 5.  Isso significa que `Array.isArray(v)` retorna `false` .  Um valor JavaScript é convertido em uma matriz do Windows Runtime da seguinte forma: se o valor for ou , ele `null` será convertido em um nativo `undefined` `null` .  Se sua propriedade interna for "Array", ela será copiada para uma matriz nativa e uma referência a `[[Class]]` essa matriz será passada.  Se for uma matriz convertida, conforme descrito anteriormente, a matriz nativa subjacente será passada.  **OBSERVAÇÃO**: passar um valor de matriz JavaScript para um método do Windows Runtime causa uma cópia completa da matriz.  |  
| Delegar \(retorno de chamada de função\) | `Function` | Um representante do Windows Runtime é uma referência a um único método.  Um representante do Windows Runtime é representado em JavaScript como um JavaScript , que é garantido para `Function` ser chamado no thread apropriado.  Quando o `Function` é invocado, os argumentos são convertidos para os tipos de parâmetros equivalentes do Windows Runtime.  Se houver menos argumentos JavaScript do que `in` parâmetros de representante, a chamada delegada falhará.  Se houver mais argumentos JavaScript do que parâmetros no representante, os `in` argumentos JavaScript extras serão ignorados.  Depois que o representante é invocado, `out` os parâmetros são convertidos em tipos JavaScript e retornados.  Se um objeto de função JavaScript nativo for convertido em um representante do Windows Runtime, ele será envolvido em um representante do Windows Runtime do tipo correspondente.  Quando o representante é invocado, os `in` parâmetros são convertidos em tipos JavaScript.  Depois que o representante é invocado, o valor de retorno é convertido para `out` os parâmetros do representante.  |  
| Interface | `Object` | Interfaces e grupos de interface não são representados diretamente em JavaScript como objetos.  No entanto, as interfaces são representadas como parâmetros e retornam tipos de métodos do Windows Runtime.  Uma instância de interface do Windows Runtime é convertida da seguinte forma:<br /> <br /> 1. Se houver uma classe de tempo de execução válida, represente o objeto como uma instância dessa classe.<br /> 2. Se não houver uma classe de tempo de execução válida, represente o objeto como uma instância de uma classe de tempo de execução sem nome que implementa exatamente a interface que a instância é conhecida por implementar, além de suas interfaces necessárias.<br /> <br /> Um valor JavaScript é testado para determinar se é uma instância de uma classe de tempo de execução ou uma interface.  Se for, e o valor original do Windows Runtime implementar a interface, o objeto será passado para o Windows Runtime.  |  
| Classes de tempo de execução | `Object` | Os objetos no Windows Runtime podem ser instâncias de classes de tempo de execução que implementam uma ou mais interfaces.  Os objetos do Windows Runtime são representados em JavaScript como objetos.  A união de métodos, propriedades e eventos definidos em todas as interfaces implementadas da classe representa as propriedades nomeadas no protótipo do objeto JavaScript.  Os consumidores JavaScript podem acessar qualquer membro da classe diretamente.  Os objetos do Windows Runtime têm um protótipo que contém todos os membros de todas as interfaces implementadas da classe de tempo de execução.  |
